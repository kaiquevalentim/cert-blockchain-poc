# 19. Verificador de Integridade de Blockchain para Certidões

## 19.1 Objetivo de Pesquisa

Desenvolver um sistema que explique em linguagem cidadã o status de integridade de certidões registradas em blockchain.


## 19.2 Componentes Técnicos / Inovadores

LLM para análise de logs de transações blockchain;
Hyperledger Fabric para registro imutável;
Dashboard explicativo acessível a operadores e cidadãos.


## 19.3 Metodologia de Desenvolvimento

Configuração de rede blockchain para certidões;
Registro de amostras de certidões digitais;
Desenvolvimento de módulo de análise automática de logs;
Geração de relatórios explicativos em linguagem natural;
Testes com operadores de cartório e cidadãos.


## 19.4 Métricas de Avaliação

Clareza dos relatórios gerados;
Tempo de verificação por documento;
Confiabilidade percebida pelos usuários.


## 19.5 Validação Experimental

Comparação entre verificações técnicas e relatórios simplificados;
Estudos de usuário para medir compreensão.


## 19.6 Resultados Esperados

Aumento da transparência do uso de blockchain;
Explicabilidade para cidadãos leigos.


## 19.7 Aspectos Éticos e Jurídicos

Garantia de integridade e privacidade nas transações;
Evitar dependência excessiva de explicações automáticas.


## 19.8 Recomendações de Implementação

Stack: Hyperledger Fabric + GPT-4/Claude + Streamlit;
Dataset: registros blockchain simulados.


## 19.9 Critérios de Avaliação Acadêmica

Contribuição para explicabilidade em blockchain;
Potencial de publicação em conferências de segurança digital.
